Begint met een unsorted-idx op -1 lengte. En een inner-idx op 0 deze dient om tot de unsorted-idx te gaan.
De unsorted-idx dient om te lopen tot waar het nog niet gesorteerd is. ^8MBhoqXF

Bubble sort ^6fhR5cYV

We selecteren 2 elementen op posities inner-idx en inner-idx + 1.
5, 6. De elementen staan relatief juist dus incrementen we de inner-idx. ^EXzzU869

De elementen 6, 1 zijn relatief niet juist dus swappen we ze. ^Iiff6Ha2

6, 3 weer niet relatief dus terug swappen ^jNHFe6HN

Op de laatste stap van de iteratie zit de inner op de unsorted idx dus zullen we de 
indices verzetten, inner-idx op 0 en de unsorted-idx -1 doen. ^Dpbi9xDc

In de laatste stap zijn de 1, 0 nog van plaats veranderd en stopt het algoritme ^zDUTXCZe

Elke iteratie het grootste element naar achter 
bubbelen. ^5kTMNLpS

Performantie ^na2Evbn1

Best-case: inner loop 1 keer en has-changed op #f 
dus r(n) = 1 met n-1 compares en 0 swaps.

Worst-case: er zijn n-1 iteraties, elke keer wordt has-changed
op #t gezet dus r(n) = n - 1 met n kwadraat compares en swaps ^c8laUE4A

Elke iteratie wordt er ook een has-changed op #f gezet als deze op het einde 
van een iteratie niet op #t staat dan kan het algoritme vroeger stoppen. ^G0EBhXk5

Eigenschappen ^2fe0BuQx

- Stabiel tenzij strikte ongelijkheid
- Gemakkelijk voor gelinkte lijsten
